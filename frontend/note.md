身為資料分析師，我將你需要收集的資訊拆解成三大模組：**A. 基本輪廓**、**B. 節奏與風格**、**C. 主題與體驗**。每個維度都對應了即將餵給 LLM 的「系統標籤（Tags）」，這能確保 LLM 產出的結果不會發散，並且完全符合「捷運可達」的隱藏條件。

以下是為你統整的 **10 個旅遊偏好面向（9 題外顯題 + 1 個系統預設）**：

### 📊 客製化旅遊偏好面向總表

#### **模組 A：基本旅遊輪廓 (Profile & Constraints)**
*這部分用於確立行程的「硬性限制條件」，避免 LLM 推薦不適合的餐廳或無法負擔的行程。*

| 題號 | 偏好維度 | 測驗核心目的 | 對應的系統標籤 (餵給 LLM 的 Data Tags) |
| :--- | :--- | :--- | :--- |
| **Q1** | **旅伴結構** | 決定行程體力負荷、是否需要無障礙/育嬰設施 | `獨旅`、`情侶/夫妻`、`朋友群體`、`親子(帶小孩)`、`帶長輩` |
| **Q2** | **預算等級** | 決定餐飲、購物商圈與付費展演的推薦等級 | `小資窮遊(高CP值)`、`中等預算(舒適)`、`奢華享受(無預算上限)` |
| **Q3** | **飲食限制** | 確保夜市/餐廳推薦安全（排除地雷） | `無限制`、`素食/鍋邊素`、`不吃牛`、`海鮮過敏`、`其他(開放填空)` |

#### **模組 B：行程節奏與風格 (Pace & Vibe)**
*這部分用於確立行程的「骨架」，決定一天的景點數量與場域分配。*

| 題號 | 偏好維度 | 測驗核心目的 | 對應的系統標籤 (餵給 LLM 的 Data Tags) |
| :--- | :--- | :--- | :--- |
| **Q4** | **行程緊湊度** | 決定一天要塞幾個景點 (Pace) | `漫遊放鬆(1-2個)`、`充實適中(3-4個)`、`行軍打卡(5個以上)` |
| **Q5** | **室內外比例** | 依據天氣或個人喜好決定場域屬性 | `全室內(吹冷氣/避雨)`、`室內外平均交錯`、`全室外(擁抱自然/街景)` |
| **Q6** | **時代氛圍** | 決定景點的文化調性 (Vibe) | `懷舊復古(老街/古蹟)`、`新創現代(摩登/潮流)`、`兩者皆可(對比體驗)` |

#### **模組 C：主題與體驗偏好 (Themes & Activities)**
*這部分用於填補行程的「血肉」，這裡會大量運用你提到的各項主題。為了讓 LLM 更好抓重點，我們將體驗分為日間、夜間與核心必備。*

| 題號 | 偏好維度 | 測驗核心目的 | 對應的系統標籤 (餵給 LLM 的 Data Tags) |
| :--- | :--- | :--- | :--- |
| **Q7** | **日間主要活動** | 決定白天的核心行程 (建議設為複選，最多選2項) | `商圈購物`、`市集展演`、`文青散策(咖啡廳/巷弄)`、`親子體驗(手作/樂園)`、`秘境探索(捷運可達之隱藏景點)` |
| **Q8** | **夜間生活安排** | 決定晚上的行程亮點 (建議設為複選，最多選2項) | `夜間美食(夜市/街邊小吃)`、`演唱會/展演活動`、`微醺夜生活(酒吧/餐酒館)`、`夜景散步` |
| **Q9** | **行程絕對亮點** | 強制使用者選出「這次旅行最不能妥協的 1 個重點」，讓 LLM 作為首要權重 | `吃爆在地美食`、`買到手軟`、`拍出完美美照`、`深度文化體驗`、`徹底放鬆身心` |

#### **⚙️ 模組 D：系統隱藏設定 (System Prompt - 不顯示於問卷)**
| 題號 | 偏好維度 | 系統強制指令 (System Prompt for LLM) | 
| :--- | :--- | :--- |
| **-** | **交通方式** | `強制條件`：所有推薦景點與行程，**必須是「捷運步行 15 分鐘內可達」的地點**。不可安排需要開車或包車的深山/郊區秘境。 |

### 💡 資料分析師的實務建議：
1. **多選題的限制**：在 Q7 和 Q8（主題活動）中，建議限制使用者**「最多選 2-3 項」**。如果讓他們全選，LLM 產出的行程會失去焦點，變成大雜燴。
2. **Q9 的重要性**：Q9 是一個「權重題（Weighting）」。當使用者的偏好發生衝突（例如選了「漫遊放鬆」又選了超多「主題活動」）時，LLM 可以根據 Q9 的最高指導原則來做取捨。
